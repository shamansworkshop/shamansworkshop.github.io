About Sand Box 3D

What is Sand Box 3D?

Sand Box 3D is a fully interactive 3D physics sandbox built with Babylon.js and Cannon.js. It provides a creative playground where you can spawn objects, apply physics forces, freeze objects in place, and even cause spectacular explosions. Perfect for experimentation, stress testing, or just having fun with physics!

Core Philosophy

science Physics-Driven

Realistic physics simulation with gravity, collisions, and forces

brush Creative Freedom

Build structures, create chaos, or test physics scenarios

control_camera Full Control

Fly freely or possess objects for direct control

touch_app Mobile Ready

On-screen controls for mobile devices

Key Features

  • categoryMultiple Shapes: Spawn cubes, spheres, cylinders, and triangular prisms
  • paletteColor Customization: Choose any color for spawned objects
  • straightenSize Control: Adjust object scale from 0.2x to 5.0x
  • gamepadInteractive Tools: Push, freeze, erase, and possess objects
  • constructionBuild Mode: Connect objects together with physics joints
  • boltExplosives: Standard bombs and cluster bombs for destruction
  • visibilityPossess Mode: Take control of any physics object
  • terrainPre-built Environment: Includes ground, ramp, and decorative pillars

Technical Architecture

  • language3D Engine: Babylon.js (WebGL-based rendering)
  • memoryPhysics Engine: Cannon.js (rigid body dynamics)
  • phone_androidResponsive Design: Touch controls for mobile devices
  • cleaning_servicesKill Zone: Auto-cleanup for objects falling below Y = -100

Tools & Actions

None (Safe Mode) üõ°Ô∏è SAFE

Disables all tools and spawning. Use this to explore the scene without accidentally creating objects or modifying the environment.

Possess Object üëÅÔ∏è VIEW

Click on any physics-enabled object to take control of it. When possessing an object:

  • gamepadCamera follows the object from above
  • arrow_forwardWASD applies force to move the object
  • exit_to_appClick "Possess Object" again to exit control

Force Push üíö PUSH

Click on any movable object to apply a powerful impulse force in the direction of your camera ray. Great for launching objects or creating chain reactions!

Eraser üóëÔ∏è DELETE

Click on objects to instantly delete them. Cannot erase ground or ramp (protected static objects). Useful for cleaning up clutter.

Freeze / Unfreeze ‚ùÑÔ∏è FREEZE

Toggle between freeze and thaw modes:

  • ac_unitFreeze: Click objects to lock them in place (mass = 0, icy blue appearance)
  • whatshotUnfreeze: Click frozen objects to restore their original mass and color

Build Mode üîó JOIN

When enabled, newly spawned objects will be physically connected to the object you click on using physics joints. Perfect for building structures, vehicles, or contraptions!

Explosives üí• BOOM

Bomb (Standard)

A timed explosive that detonates after 4 seconds. Features:

  • timer4-second countdown with red flashing warning
  • auto_awesomeCreates expanding explosion sphere effect
  • blur_on25-unit blast radius (scaled by bomb size)
  • whatshotApplies radial force to all nearby objects

Cluster Bomb (Frag)

Advanced explosive that spawns shrapnel fragments. Features:

  • timer4-second countdown with orange flashing warning
  • scatter_plotSpawns 20 small physics-enabled fragments
  • whatshotFragments are launched in all directions
  • auto_awesomeMain explosion + fragment secondary effects

warning Explosive Safety

  • reportExplosives cannot be frozen or connected in Build Mode
  • reportCluster bombs can significantly increase object count
  • reportMultiple explosions may cause performance lag

  • dark_modeGround Color: Dark grey (0.15, 0.15, 0.2 RGB)

Kill Zone System

Objects that fall below Y = -100 are automatically removed from the scene. This prevents infinite falling and keeps performance stable.

info Performance Tip

If you're experiencing lag, use the CLEAR ALL button to remove all spawned objects and reset the scene.

Known Limitations & Issues

info Current Limitations

  • sports_esportsNo Save/Load: Scene state is not persistent between sessions
  • downloadNo Export: Cannot export created structures or scenes
  • undoNo Undo/Redo: Actions cannot be reversed (use CLEAR ALL to reset)
  • content_copyNo Clone Tool: Cannot duplicate existing objects
  • textureBasic Materials: No texture support, only solid colors

Performance Considerations

  • trending_downFrame rate may drop with 200+ objects in scene
  • trending_downMultiple explosions can cause temporary lag spikes
  • trending_downBuild Mode joints with many connections may become unstable
  • trending_downMobile devices have lower object count limits (~100 objects)

Troubleshooting

Issue Solution
Black screen / Not loading Check WebGL support, try different browser
Severe lag / Low FPS Click CLEAR ALL, reduce object count

For Developers

Code Structure

Sand Box 3D is organized into two main sections:

  • dashboardUI Logic (Section 1): Panel controls, button handlers, mobile inputs
  • view_in_arBabylon Engine (Section 2): 3D scene, physics, rendering, interaction

Key Variables

Variable Purpose
selectedShape Current tool/shape selection
spawnedMeshes[] Array of all spawned objects
isBuildMode Build Mode toggle state
viewTargetMesh Currently possessed object
currentSize Spawn size multiplier
freezeAction 'freeze' or 'thaw' mode

Customization Tips

  • tuneModify Physics: Adjust gravity, friction, restitution in createScene()
  • add_circleAdd Shapes: Create new mesh types in spawnObject() function
  • color_lensChange Theme: Edit CSS variables in :root
  • wb_sunnyAdjust Lighting: Modify HemisphericLight and DirectionalLight values
  • landscapeCustom Environments: Add terrain in createScene()

Extension Points

Developers can extend Sand Box 3D by:

  • webhookAdding new tools to the setupInteraction() function
  • categoryCreating custom shape generators in spawnObject()
  • animationImplementing new visual effects in the render loop
  • saveAdding save/load functionality using localStorage
